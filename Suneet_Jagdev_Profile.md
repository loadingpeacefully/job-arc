### Project 3: PQS — Platform Quality Score (Pedagogical Quality System)

Suneet diagnosed that educational quality assurance was manual, subjective, and trapped in backend silos — invisible to parents. Redesigned the entire system.

**Architecture:**

- **Layer 1 — Data Ingestion & Routing** — captures class transcripts across Zoom and Google Meet; enforces >75% English threshold; flags non-English sessions (Bahasa, Hindi, Bangla) for localised prompt pipeline; currently covers 70% of all 1:1 paid class volume
- **Layer 2 — Core Engine (Forensic Ledger)** — LLM evaluates transcripts against a 10-point pedagogical rubric (Concept Clarity, Worked Examples, Guided Practice, Homework Check, Student Comprehension etc.); generates a structured JSON array mapping micro-events to exact timestamps with speaker role, event type tags, and verbatim evidence quotes
- **Layer 3 — Narrative Generation (Parent-Facing)** — translates JSON ledger into an interactive HTML parent report; visualises minute-by-minute "Learning Journey"; quantifies "Class Energy" (teacher/student talk-time ratio); generates context-aware parent prompts for at-home reinforcement

**Key design insight:** Quality assurance as a retention engine, not just an ops tool. The parent report turns backend data into a product feature parents see.

**Outcomes:** 91% of all completed paid classes scored accurately. 79% total PQS generation rate (including cancelled/incomplete sessions).

**Roadmap:** Multi-lingual expansion → Schola/Adult English → Google Meet audio pipeline → dynamic parent report UI.
